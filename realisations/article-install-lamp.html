Préparation de l'infrastructure

La première étape a consisté à configurer l'environnement réseau de la machine virtuelle. J'ai modifié la configuration IP pour passer d'un adressage dynamique à une IP statique via le fichier interfaces situé dans /etc/network. Afin de faciliter la saisie des commandes et le transfert de code, je me suis connecté à la VM via SSH en utilisant OpenSSH.

Une mise à jour complète du système a également été effectuée, faisant passer la distribution de Debian 12.0 à Debian 13.2.

Installation et test des services

  • Apache2 : Après l'installation, le bon fonctionnement du serveur web a été vérifié en accédant à l'adresse IP de la VM (192.168.12.2).
  • PHP : J'ai installé PHP ainsi que sa librairie pour Apache2. La validation a été faite via la création d'une page info.php.
  • MariaDB : Suite à l'installation, j'ai exécuté la commande de sécurisation initiale et vérifié l'accès au service.

Développement et base de données

Une fois les services opérationnels, j'ai créé une base de données nommée tptest ainsi qu'un utilisateur dédié. Pour tester l'interaction entre les composants, j'ai développé plusieurs scripts PHP :

  • Un script de vérification de connexion à la base de données. [cite: 9]
  • Les fichiers formulaire.php et liste.php pour tester l'insertion et l'affichage des données en temps réel.

Analyse et résolution de problèmes

Problème rencontré : Lors du test de connexion à la base de données, la page PHP ne s'ouvrait pas correctement.

Solution : L'absence du pilote de liaison entre PHP et MySQL était en cause. L'exécution de la commande apt install php8.4-mysql a permis de rétablir la communication entre les services.
